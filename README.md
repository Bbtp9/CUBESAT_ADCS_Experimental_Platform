# CubeSat ADCS Experimental Platform

Bachelor's Thesis Project
Aerospace Engineering

## Project Overview

This project focuses on the design and experimental validation of an **Attitude Determination and Control System (ADCS)** for a **1U CubeSat**.

The work combines:

* spacecraft system design
* attitude determination algorithms
* attitude control using magnetorquers
* MATLAB / Simulink simulation
* real-time implementation on a Raspberry Pi
* an experimental hardware demonstrator

The goal is to demonstrate **attitude control around one axis** using magnetorquers on a suspended CubeSat platform.

---

## Mission Concept

The reference mission is an **Earth observation CubeSat** placed in a **Sun-Synchronous Orbit (SSO) at 550 km altitude**.

The payload camera is oriented toward Earth to monitor **cloud formations and atmospheric patterns**.

Mission objectives:

* maintain nadir pointing
* acquire Earth images
* demonstrate CubeSat attitude control

## Satellite Configuration

Satellite type: **1U CubeSat**

Dimensions:

```
10 cm × 10 cm × 10 cm
```

Main subsystems:

### Structure

* 3D printed CubeSat frame
* lightweight PLA structure
* mounted inside a laboratory frame

### Onboard Computer (OBC)

* Raspberry Pi Zero 2 W
* runs control algorithms
* processes sensor data
* commands magnetorquers

### Sensors

Attitude estimation uses:

* IMU (gyroscope + accelerometer)
* magnetometer
* Sun sensor
* horizon detection camera

### Actuators

Attitude control is performed using **magnetorquers**.

They generate magnetic dipole moments interacting with Earth's magnetic field.

Control torque:

```
T = m × B
```

where

* m = magnetic dipole generated by the coils
* B = Earth's magnetic field

## Attitude Determination

Attitude is estimated using:

* magnetometer measurements
* Sun sensor measurements
* horizon camera

Possible algorithms:

* TRIAD
* q-method
* quaternion estimation

---

## Attitude Control

The spacecraft attitude is controlled using a **Linear Quadratic Regulator (LQR)** controller.

State vector:

```
x = [attitude error, angular velocity]
```

Control law:

```
u = −Kx
```

where **K** is the optimal gain matrix computed from the Riccati equation.

The controller outputs the **desired magnetic dipole moment**, which is converted to coil current.

## Simulation Environment

Before hardware implementation, the system is simulated in **MATLAB / Simulink**.

The simulation includes:

* spacecraft rotational dynamics
* Earth's magnetic field model
* sensor models
* control algorithm

This allows validation of the control law before deploying it on hardware.

---

## Experimental Platform

To experimentally demonstrate the control concept, a **laboratory testbed** is used.

The CubeSat mock-up is:

* suspended from a long wire
* free to rotate around one axis
* equipped with magnetorquer coils

The control system runs on the Raspberry Pi and commands the magnetorquers to rotate the CubeSat to a desired orientation.

This setup allows demonstration of real ADCS behavior.
